# nw_wrld VJ Project

## Overview
Code-driven VJing and data visualization platform using nw_wrld software sequencer. Enables real-time visual performances driven by data feeds, MIDI/OSC triggers, and creative coding.

## Purpose
Create live visual performances and installations that:
- React to audio and music
- Visualize real-time data streams
- Integrate system and agent metadynamics
- Support physical installations and projection mapping
- Enable creative coding workflows

## Technologies
- nw_wrld event-driven software sequencer
- HTML, CSS, JavaScript
- WebGL and Three.js for 3D graphics
- OSC (Open Sound Control) for data integration
- MIDI for performance control
- Raspberry Pi compatible for installations

## Planned Data Integrations
- **System/Agent Metadynamics** - Integration with Executive Mind Matrix
- **Audio Reactivity** - FFT analysis for music visualization
- **Financial Data** - Real-time sector flows and blockchain activity
- **Network Traffic** - System activity visualization
- **Environmental Sensors** - Physical space reactivity

## Key Features
- Custom visual module development
- Live VJ setup with MIDI/OSC triggering
- Real-time data streaming and visualization
- Projection mapping capabilities
- Ambient display modes
- Environmental sensor integration

## Project Structure
- `/modules/` - Custom visual modules
- `/assets/` - Visual assets and textures
- `/nw_wrld_data/` - Configuration and data files
- Comprehensive documentation (OSC setup, shader guides, workflow)

## Integration Points
- **Executive Mind Matrix** - Visualize system metadynamics
- **Sector Flow Analyzer** - Real-time financial flow visualization
- **Memecoin Sentiment Analyzer** - Social sentiment visualization
- **Observability Stack** - System metrics visualization

## Current Status
Planning and prototyping phase. Documentation and setup guides in development.

## Physical Installation Capabilities
- Projection mapping support
- Ambient display configurations
- Raspberry Pi deployment for standalone installations
- Environmental sensor reactivity

## Key Decisions
- Chose nw_wrld over TouchDesigner for code-first workflow and cost
- Selected OSC for data integration due to flexibility and real-time capability
- Planned multi-source data integration from existing projects
- Focused on both live performance and installation use cases
- Raspberry Pi compatibility for portable/permanent installations
