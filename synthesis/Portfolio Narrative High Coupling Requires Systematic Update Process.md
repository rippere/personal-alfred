# Portfolio Narrative High Coupling Requires Systematic Update Process

## Insight
Career portfolio narrative documents are tightly coupled — a single positioning change (e.g., updating academic major/minor) required simultaneous updates to 7+ files plus associated vault records. This coupling is structural: each document encodes the same positioning in a different context (cover letters, discovery questions, company targets, checklists), so any positioning shift ripples across all of them.

## Evidence
The Major Change Update Session updated all of the following for a single positioning change (neuroscience → psychology major):
- PORTFOLIO_NARRATIVE.md
- CAREER_ROADMAP.md
- JOB_SEARCH_README.md
- INDEX.md
- APPLICATION_TEMPLATES.md
- COMPANY_TARGETS.md
- DISCOVERY_QUESTIONS.md
- APPLICATION_CHECKLIST.md
- person/Benjamin Ripperie.md (vault record)
- project/Career Portfolio Narrative.md (vault record)

A PM agent was deployed specifically to handle the systematic update load, indicating the scope exceeded what a single-file edit could address.

## Implications
Future positioning changes (new job target, new project emphasis, major academic or career update) will require an equivalent systematic sweep. Ad-hoc updates risk leaving stale positioning in some files while others are current, creating internal inconsistency that surfaces during applications.

## Applicability
Any future career narrative evolution for the Career Portfolio Narrative project. The pattern suggests maintaining an explicit "portfolio update checklist" as a standard change-management artifact whenever core positioning shifts.
